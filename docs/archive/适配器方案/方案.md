# DeepSeek 适配 Codex 协议转换：Cloudflare Worker 最小适配器方案

---
生成日期：2026-06-19
目标：在 Cloudflare Worker 内实现 Responses API → Chat Completions 的最小协议转换层，使 Codex Desktop / CLI 能以 `wire_api = "responses"` 直连 DeepSeek（经 AI Gateway 或直连官方 API），无需 Container / OpenResponses。
关联文档：`docs/方案.md`（Container 路线）、`docs/方案挑战.md`（架构挑战评审）
---

## 0. 推荐结论

个人使用第一版建议采用：

```text
Codex Desktop / CLI
  → Cloudflare Worker（鉴权 + Responses↔Chat 协议适配）
  → Cloudflare AI Gateway DeepSeek provider-native（可选）
  → DeepSeek 官方 API（你的 DeepSeek Key）
```

**与 Container 路线的核心差异：** 协议转换在 Worker TypeScript 中完成，不跑 Docker / OpenResponses，无冷启动。

**推荐目标层级：Tier 2**（流式 + 客户端工具调用 + reasoning 映射），预计 1–2 周全职或 3–4 周业余可落地。

---

## 1. 为什么选 Worker 适配器

| 维度 | Container + OpenResponses | Worker 最小适配器 |
|---|---|---|
| 冷启动 | 有（sleep 后 10–60s+） | 无 |
| 部署依赖 | Docker + Container + DO | 仅 `wrangler deploy` |
| 月成本 | Workers Paid + Container 活跃时长 | Workers Paid（~$5） |
| 协议兼容 | 社区 OpenResponses 维护 | 自行维护 Tier 2 子集 |
| 适合场景 | 快速零开发上线 | 愿投入 1–2 周、讨厌冷启动 |

**成立前提（与 Container 方案相同）：**

1. Codex custom provider 必须 `wire_api = "responses"`。
2. DeepSeek 仅提供 OpenAI-compatible Chat Completions，无原生 `/v1/responses`。
3. 自有 DeepSeek Key 不能走 Cloudflare REST `/ai/v1/responses`（该路径用 Cloudflare Token + Unified Billing）。

## 2. 范围界定：「最小」指什么

### 2.1 必须实现

| 端点 / 能力 | 说明 |
|---|---|
| `POST /v1/responses` | 主路径，`stream=true` 优先 |
| `GET /v1/models` | 返回 DeepSeek 模型列表（可静态配置） |
| `GET /healthz` | 健康检查 |
| 流式纯文本 | `response.created` → `output_text.delta` → `response.completed` |
| 客户端工具调用 | Codex 本地执行 shell/读文件；适配器只转换协议，**不执行工具** |
| `function_call_output` 回合 | 将工具结果写入 Chat `role=tool` messages |
| `instructions` | 映射为 system message |
| `reasoning.effort` | 映射为 DeepSeek `reasoning_effort` + `thinking.type` |
| SSE heartbeat | 长推理阶段每 15s 发送 `: heartbeat\n\n` |

### 2.2 第一版明确不做

| 能力 | 原因 |
|---|---|
| MCP 服务端执行 | Codex 工具由客户端执行 |
| 服务端 tool loop（25 轮） | 由 Codex 驱动多轮请求 |
| Hosted tools（web search、code interpreter 等） | 非 Codex 本地工具链 |
| WebSocket | Codex 配置 `supports_websockets = false` |
| `previous_response_id` 持久化 | Tier 2 先无状态；抓包确认后再加 DO KV |
| 完整 OpenResponses 事件全集 | 仅覆盖 Codex 实测用到的子集 |

### 2.3 复杂度分层

| 层级 | 能力 | 代码量 | 工期 | 难度 |
|---|---|---|---|---|
| Tier 0 | 非流式 hello world | ~100 行 | 0.5 天 | ★☆☆☆☆ |
| Tier 1 | 流式纯文本 | ~400–600 行 | 2–4 天 | ★★☆☆☆ |
| **Tier 2（推荐）** | + 工具调用 + reasoning | ~800–1,500 行 | 1–2 周 | ★★★☆☆ |
| Tier 3 | + DO 会话 + 全量兼容 | ~2,000–3,000+ 行 | 4–8 周 | ★★★★★ |

OpenResponses 参考：`responses_service.py` 约 920 行，其中流式状态机 ~600 行。Tier 2 对标其 **Codex 客户端工具路径**，不移植 MCP / 服务端 loop。

## 3. 总体架构

```text
Codex Desktop / Codex CLI
  │
  │ POST /v1/responses
  │ Authorization: Bearer CODEX_GATEWAY_API_KEY
  ▼
Cloudflare Worker
  ├─ 鉴权（CODEX_GATEWAY_API_KEY）
  ├─ 路径白名单（/v1/responses, /v1/models, /healthz）
  ├─ adapters/request.ts   Responses body → Chat Completions body
  ├─ adapters/stream.ts    Chat SSE chunks → Responses SSE events
  └─ deepseek.ts           上游 fetch（Gateway 或直连）
  ▼
（可选）Cloudflare AI Gateway provider-native
  https://gateway.ai.cloudflare.com/v1/{ACCOUNT_ID}/{GATEWAY_ID}/deepseek/chat/completions
  Authorization: Bearer DEEPSEEK_API_KEY
  ▼
DeepSeek official API
  model = deepseek-v4-pro / deepseek-v4-flash
```

**Codex 工具调用时序（客户端执行，适配器无状态转发）：**

```text
1. Codex → POST /v1/responses { tools, input }
2. Worker → DeepSeek chat/completions (stream)
3. Worker ← tool_calls delta
4. Worker → Codex SSE: function_call (status=ready) + response.completed
5. Codex 本地执行 shell/read 等
6. Codex → POST /v1/responses { input: [function_call_output, ...] }
7. Worker 将 output 转为 role=tool message，继续请求 DeepSeek
8. 重复直至 finish_reason=stop
```

## 4. 核心难点与对策

### 4.1 流式 SSE 状态机（最高风险）

Chat Completions SSE 需映射为 Responses API 事件序列。以 [OpenResponses Events 文档](https://open-responses-server.teabranch.dev/docs/events-and-tool-handling.html) 为 golden reference。

| 上游 (Chat Completions) | 下游 (Responses SSE) |
|---|---|
| （流开始） | `response.created` |
| （流开始） | `response.in_progress` |
| `delta.content` | `response.output_text.delta` |
| `delta.tool_calls[].function.name`（首次） | `response.in_progress`（output 含 function_call） |
| `delta.tool_calls[].function.arguments` 片段 | `response.function_call_arguments.delta` |
| `finish_reason=tool_calls` | `response.function_call_arguments.done`（每个 tool） |
| `finish_reason=tool_calls` | `response.completed`（function_call status=ready） |
| `finish_reason=stop` | `response.completed` |
| `[DONE]` 且无 completed | 补发 `response.completed` |

**对策：** 为 5 种 scenario 各写 fixture 测试；开发时对照 OpenResponses 日志。

### 4.2 会话状态

| 策略 | 做法 | 适用 |
|---|---|---|
| A. 无状态（Tier 2 默认） | 假定 Codex 在 `input` 中带全量上下文 | 抓包验证通过则足够 |
| B. DO KV（Tier 2+） | `previous_response_id` → 存 messages[] | Codex 依赖服务端历史时启用 |

**验证方法：** `wrangler tail` 记录前 3 次真实 Codex 请求，检查是否仅依赖 `previous_response_id`。

### 4.3 `function_call_output` 拼接

当 `input` 含 `function_call_output` 但 messages 中无 matching `tool_call_id` 时，需合成 assistant + tool_calls 消息（OpenResponses 同类逻辑 ~50 行）。缺失会导致 DeepSeek 报错：`messages with role 'tool' must be a response to...`。

### 4.4 DeepSeek reasoning 映射

Codex `model_reasoning_effort` → Responses `reasoning.effort` → Chat Completions 请求体：

```json
{
  "reasoning_effort": "high",
  "thinking": { "type": "enabled" }
}
```

`medium` / `low` 对应调整 `reasoning_effort`；`thinking.type` 可按 effort 是否非 none 启用。

### 4.5 长连接保活

- Worker → Codex：`HEARTBEAT_INTERVAL=15` 秒注入 SSE comment
- 上游 fetch：对齐 Codex `stream_idle_timeout_ms=300000`（300s）
- Worker 平台：HTTP 无 wall-clock 硬限制，`fetch` 等待不计 CPU time，适合长流式

### 4.6 长期维护

Responses API 持续演进。锁定依赖版本、维护 Codex 回归测试集、保留切换至 Container 方案的 env flag。

### 7.6 src/adapters/request.ts（核心转换骨架）

```ts
/**
 * Responses API request → OpenAI Chat Completions request.
 * Tier 2: instructions, input[], tools, stream, reasoning.effort
 */
export function responsesToChatRequest(
  req: Record<string, unknown>,
): Record<string, unknown> {
  const chat: Record<string, unknown> = {
    model: req.model,
    stream: req.stream ?? false,
    temperature: req.temperature ?? 1,
    top_p: req.top_p ?? 1,
  };

  if (req.max_output_tokens != null) {
    chat.max_tokens = req.max_output_tokens;
  }

  const messages: Array<Record<string, unknown>> = [];

  // instructions → system
  if (typeof req.instructions === "string" && req.instructions) {
    messages.push({ role: "system", content: req.instructions });
  }

  // input: string | array
  const input = req.input;
  if (typeof input === "string") {
    messages.push({ role: "user", content: input });
  } else if (Array.isArray(input)) {
    for (const item of input) {
      appendInputItem(messages, item);
    }
  }

  if (Array.isArray(req.tools) && req.tools.length > 0) {
    chat.tools = req.tools.map(normalizeTool);
  }

  // reasoning.effort → DeepSeek
  const effort = extractReasoningEffort(req);
  if (effort) {
    chat.reasoning_effort = effort;
    chat.thinking = { type: "enabled" };
  }

  chat.messages = messages;
  return chat;
}

function appendInputItem(messages: Array<Record<string, unknown>>, item: unknown) {
  if (!item || typeof item !== "object") return;
  const obj = item as Record<string, unknown>;

  if (obj.type === "message" && obj.role === "user") {
    messages.push({ role: "user", content: extractTextContent(obj.content) });
    return;
  }

  if (obj.type === "function_call_output") {
    const callId = obj.call_id as string;
    const output = (obj.output as string) ?? "";

    if (!hasMatchingToolCall(messages, callId)) {
      messages.push({
        role: "assistant",
        content: null,
        tool_calls: [{
          id: callId,
          type: "function",
          function: {
            name: (obj.name as string) ?? "unknown_tool",
            arguments: (obj.arguments as string) ?? "{}",
          },
        }],
      });
    }

    messages.push({ role: "tool", tool_call_id: callId, content: output });
  }
}

function extractTextContent(content: unknown): string {
  if (typeof content === "string") return content;
  if (!Array.isArray(content)) return "";
  return content
    .map((c) => {
      if (typeof c === "string") return c;
      if (c && typeof c === "object") {
        const o = c as Record<string, unknown>;
        if (o.type === "input_text" || o.type === "text") return String(o.text ?? "");
      }
      return "";
    })
    .join("");
}

function hasMatchingToolCall(messages: Array<Record<string, unknown>>, callId: string): boolean {
  for (const msg of messages) {
    if (msg.role !== "assistant" || !Array.isArray(msg.tool_calls)) continue;
    for (const tc of msg.tool_calls) {
      if ((tc as Record<string, unknown>).id === callId) return true;
    }
  }
  return false;
}

function normalizeTool(tool: unknown) {
  const t = tool as Record<string, unknown>;
  if (t.type === "function" && t.function) return t;
  return { type: "function", function: t };
}

function extractReasoningEffort(req: Record<string, unknown>): string | null {
  const reasoning = req.reasoning as Record<string, unknown> | undefined;
  const effort = reasoning?.effort;
  if (typeof effort === "string" && effort !== "none") return effort;
  return null;
}
```

### 7.7 src/adapters/stream.ts（状态机骨架）

```ts
/**
 * 将 DeepSeek Chat Completions SSE 转为 Responses API SSE。
 * 完整实现需覆盖：tool_calls 累积、finish_reason 分支、错误 completed。
 */
export async function* chatStreamToResponsesEvents(
  upstream: ReadableStream<Uint8Array>,
  meta: { model: string },
): AsyncGenerator<string> {
  const responseId = `resp_${crypto.randomUUID().replace(/-/g, "")}`;
  const messageId = `msg_${crypto.randomUUID().replace(/-/g, "")}`;

  const base = createBaseResponse(responseId, meta.model);
  yield sseEvent({ type: "response.created", response: base });
  yield sseEvent({ type: "response.in_progress", response: base });

  const decoder = new TextDecoder();
  const reader = upstream.getReader();
  let buffer = "";
  const toolCalls = new Map<number, { id: string; name: string; arguments: string }>();

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });

    let idx: number;
    while ((idx = buffer.indexOf("\n")) >= 0) {
      const line = buffer.slice(0, idx).trim();
      buffer = buffer.slice(idx + 1);
      if (!line.startsWith("data:")) continue;

      const payload = line.slice(5).trim();
      if (payload === "[DONE]") {
        base.status = "completed";
        yield sseEvent({ type: "response.completed", response: base });
        return;
      }

      const chunk = JSON.parse(payload);
      const choice = chunk.choices?.[0];
      if (!choice) continue;

      if (chunk.model && !base.model) base.model = chunk.model;

      const delta = choice.delta ?? {};
      if (typeof delta.content === "string" && delta.content) {
        yield sseEvent({
          type: "response.output_text.delta",
          item_id: messageId,
          output_index: 0,
          content_index: 0,
          delta: delta.content,
        });
      }

      if (Array.isArray(delta.tool_calls)) {
        for (const tc of delta.tool_calls) {
          const i = tc.index ?? 0;
          if (!toolCalls.has(i)) {
            toolCalls.set(i, {
              id: tc.id ?? `call_${crypto.randomUUID().replace(/-/g, "")}`,
              name: "",
              arguments: "",
            });
          }
          const acc = toolCalls.get(i)!;
          if (tc.function?.name) acc.name = tc.function.name;
          if (tc.function?.arguments) {
            acc.arguments += tc.function.arguments;
            yield sseEvent({
              type: "response.function_call_arguments.delta",
              item_id: acc.id,
              output_index: i,
              delta: tc.function.arguments,
            });
          }
        }
      }

      const finish = choice.finish_reason;
      if (finish === "tool_calls") {
        for (const [i, tc] of toolCalls) {
          yield sseEvent({
            type: "response.function_call_arguments.done",
            id: tc.id,
            output_index: i,
            arguments: tc.arguments,
          });
          base.output.push({
            id: tc.id,
            type: "function_call",
            status: "ready",
            name: tc.name,
            arguments: tc.arguments,
          });
        }
        base.status = "completed";
        yield sseEvent({ type: "response.completed", response: base });
        return;
      }

      if (finish === "stop") {
        base.status = "completed";
        yield sseEvent({ type: "response.completed", response: base });
        return;
      }
    }
  }
}

function sseEvent(obj: unknown): string {
  return `data: ${JSON.stringify(obj)}\n\n`;
}

function createBaseResponse(id: string, model: string) {
  return {
    id,
    object: "response",
    created_at: Math.floor(Date.now() / 1000),
    status: "in_progress",
    model,
    output: [] as unknown[],
  };
}
```

### 7.8 src/utils/heartbeat.ts

```ts
export function withHeartbeat(
  stream: ReadableStream<Uint8Array>,
  intervalMs: number,
): ReadableStream<Uint8Array> {
  const encoder = new TextEncoder();
  const heartbeat = encoder.encode(": heartbeat\n\n");
  let timer: ReturnType<typeof setInterval> | undefined;

  return new ReadableStream({
    start(controller) {
      const reader = stream.getReader();
      timer = setInterval(() => {
        try { controller.enqueue(heartbeat); } catch { /* closed */ }
      }, intervalMs);

      (async () => {
        try {
          while (true) {
            const { value, done } = await reader.read();
            if (done) break;
            controller.enqueue(value);
          }
          controller.close();
        } catch (e) {
          controller.error(e);
        } finally {
          if (timer) clearInterval(timer);
        }
      })();
    },
    cancel() {
      if (timer) clearInterval(timer);
    },
  });
}
```

### 7.9 src/index.ts

```ts
import type { Env } from "./env";
import { postChatCompletions } from "./deepseek";
import { responsesToChatRequest } from "./adapters/request";
import { chatStreamToResponsesEvents } from "./adapters/stream";
import { withHeartbeat } from "./utils/heartbeat";

function unauthorized() {
  return Response.json({ error: { message: "Unauthorized" } }, { status: 401 });
}

function isAllowedPath(pathname: string) {
  return pathname === "/healthz" ||
         pathname === "/v1/models" ||
         pathname === "/v1/responses" ||
         pathname.startsWith("/v1/responses/");
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url);
    if (!isAllowedPath(url.pathname)) {
      return new Response("Not found", { status: 404 });
    }

    if (url.pathname === "/healthz") {
      return new Response("ok", { status: 200 });
    }

    const auth = request.headers.get("authorization") ?? "";
    if (auth !== `Bearer ${env.CODEX_GATEWAY_API_KEY}`) {
      return unauthorized();
    }

    if (url.pathname === "/v1/models" && request.method === "GET") {
      const models = env.SUPPORTED_MODELS.split(",").map((id) => id.trim()).filter(Boolean);
      return Response.json({
        object: "list",
        data: models.map((id) => ({ id, object: "model", created: 0, owned_by: "deepseek" })),
      });
    }

    if (url.pathname === "/v1/responses" && request.method === "POST") {
      const body = await request.json<Record<string, unknown>>();
      const chatBody = responsesToChatRequest(body);
      const timeoutMs = Number(env.STREAM_TIMEOUT_MS) || 300_000;
      const signal = AbortSignal.timeout(timeoutMs);

      const upstream = await postChatCompletions(env, chatBody, signal);
      if (!upstream.ok) {
        const errText = await upstream.text();
        return new Response(errText, {
          status: upstream.status,
          headers: { "content-type": upstream.headers.get("content-type") ?? "application/json" },
        });
      }

      if (chatBody.stream) {
        const events = chatStreamToResponsesEvents(
          upstream.body!,
          { model: String(chatBody.model ?? "") },
        );
        const sseStream = readableFromAsyncGenerator(events);
        const heartbeatMs = Number(env.HEARTBEAT_INTERVAL_MS) || 15_000;
        return new Response(withHeartbeat(sseStream, heartbeatMs), {
          headers: {
            "content-type": "text/event-stream",
            "cache-control": "no-cache",
            connection: "keep-alive",
          },
        });
      }

      // 非流式：Tier 1 可在此组装 JSON response；Tier 2 优先流式
      return upstream;
    }

    return new Response("Method not allowed", { status: 405 });
  },
};

function readableFromAsyncGenerator(gen: AsyncGenerator<string>): ReadableStream<Uint8Array> {
  const encoder = new TextEncoder();
  return new ReadableStream({
    async pull(controller) {
      const { value, done } = await gen.next();
      if (done) { controller.close(); return; }
      controller.enqueue(encoder.encode(value));
    },
  });
}
```

## 11. 排错表

| 现象 | 常见原因 | 处理 |
|---|---|---|
| Codex 401 | 本地 key 与 Worker secret 不一致 | 重设 env 与 `wrangler secret` |
| 上游 401 | DeepSeek key 错误 | 检查 secret |
| 流式秒断 | 无 heartbeat / 上游超时 | 检查 `HEARTBEAT_INTERVAL_MS`、`STREAM_TIMEOUT_MS` |
| 工具第二轮 400 | `function_call_output` 拼接错误 | 对照 §4.3，查 tail 日志中 messages |
| 无推理效果 | `reasoning.effort` 未映射 | 检查 request adapter |
| 偶发断上下文 | 依赖 `previous_response_id` | 启用 DO KV（Tier 2+） |

## 13. 与 Container 方案的切换

在 `wrangler.toml` 用不同 Worker 或环境隔离两套部署：

| 部署 | 路径 | 用途 |
|---|---|---|
| `codex-deepseek-adapter` | Worker shim | 低延迟日常 |
| `codex-deepseek-openresponses` | Container 方案 | shim 升级失败时的 fallback |

Codex profile 仅改 `base_url` 即可切换。

## 14. 安全建议

1. DeepSeek key 仅存 `wrangler secret`，Codex 本地只持 `CODEX_GATEWAY_API_KEY`。
2. Worker 入口鉴权；`/healthz` 可不鉴权（仅暴露存活状态）。
3. 路径白名单；不暴露上游错误中的敏感 header。
4. `LOG_LEVEL=INFO`，避免 DEBUG 打印完整 prompt。
5. 锁定 `wrangler`、`typescript` 版本，避免 `latest` 漂移。
